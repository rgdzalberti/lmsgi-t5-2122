Comparativa

Aquí algunas de las mejores opciones a elegir.
Estudio elaborado cuidadosamente.


OpenBravo

  • Es una aplicación completamente web. No es necesario software.
  • Generación de código automática
  • Integración personalizada y modelada
  • Tan sólo tiene soporte para BBDDs PostgreSQL y Oracle
  • No permite manejar simultáneamente varios idiomas para los clientes
  • El software es gratuito pero la implementación no es barata.
  • ELEGIR

SAP

  • Comodidad: Fácil de personalizar y usar.
  • Portabilidad: Se puede usar en diferentes sistemas operativos
  • Modularidad: Es fácil incorporarlo con software externo, en forma de módulos.
  • Complejidad: Es difícil de usar, y difícil de aprender.
  • Lento: Es un software que se tarda en incorporar dentro de la empresa.
  • Actualizaciones constantes: Lo que hace que cueste mantener módulos insertados funcionando, debidos a problemas de compatibilidad.
  • ELEGIR

Oracle

  • Fácil de Usar: El primer tier ofrece una solución 100% web, haciendo que, si se desea, se pueda ahorrar instalación de software de escritorio.
  • Fácil de Mantener: Debido a que está basado en 2 capas (tecnología y aplicación), reduce los costes de mantenimiento. Todos los cambios se aplican al servidor, y los usuarios reciben las actualizaciones automáticamente.
  • Modularidad: Es fácil incorporarlo con software externo, en forma de módulos.
  • Falta de especialización: Debido a su diseño horizontal, posee poca especialización.
  • Aumento de coste: Debido a su falta de especialización, si se desea usar para algo específico, es necesario obtener ese software hecho a medida.
  • Complejidad técnica: La instalación y actualización del software es difícil.
  • ELEGIR

Odoo

  • El precio de este software es notablemente más barato que la competencia.
  • Posibilidad de crear módulos adicionales a través de Python
  • Capacidad de reagrupar y aplicar filtros.
  • Las implantaciones en este software son muy dificultosas
  • Si necesitas muchas aplicaciones o extras puedes terminar pagando de más.
  • Necesarios conocimientos técnicos para la administración
  • ELEGIR

Openxpertya

  • Es independiente de la BD, lo cual aumenta la versatilidad del software
  • Gran seguridad
  • Existe documentación y formación disponible a todos los niveles, tanto a nivel de usuario final, como a nivel de desarrollador.
  • Es un sistema de gestión multiplataforma, está desarrollado en tres capas e íntegramente en Java, con lo que funciona
  • El sistema tiene un amplio margen de maniobra mediante extensiones del mismo que permiten añadir funcionalidades.
  • No tiene servicio técnico de atención inmediata.
  • ELEGIR

Abanq

  • Es un Sistema Multiusuario
  • Posee su propio sistema de permisos (con varios niveles para cada característica)
  • Una interfaz personalizable mediante temas
  • Permite modular el código
  • Está ideado principalmente para trabajar sobre PostgreSQL como base de datos, y, aunque permite otros controladores (como MySQL o SQLite), el único recomendado es PostgreSQL.
  • Usa tecnologías obsoletas sin soporte del fabricante
  • ELEGIR
